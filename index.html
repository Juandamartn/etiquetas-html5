<article>

Sirve para especificar contenido independiente. Es soportado por Chrome 6.0 en adelante, internet explorer/edge 9.0 en adelante, Firefox 4.0 en adelante, safari 5.0 en adelante y opera 11.1 en adelante.

Ejemplo

        
<!DOCTYPE html>
<html>
  <body>
    <article>
      <h1>Ejemplo</h1>
        <p>Este es un ejemplo de la etiqueta article</p>
    </article>
  </body>
</html>

<aside>

Sirve para definir un contenido separado del que se encuentra aunque también se puede poner como una barra lateral en el artículo. Es soportado por Chrome 6.0 en adelante, internet explorer/edge 9.0 en adelante, Firefox 4.0 en adelante, Safari 5.0 en adelante y Opera 11.1 en adelante.

Ejemplo

        
<!DOCTYPE html>
<html>
  <body>
    <p>Todos en algún momento pensamos que los ornitorrincos eran los únicos
    que nacían en huevo.</p>
    <aside>
      <h4>Nota </h4>
      <p>Existen unos animales que se llaman equidnas que también lo hacen.</p>
    </aside>
  </body>
</html>

<audio>

Esta etiqueta es para poner elementos de audio en las páginas, pero sólo admite tres tipos de audio, mp3, wav y ogg, aunque internet explorer no acepta el formato wav y ogg y safari no acepta el formato ogg, los navegadores que no soportan la etiqueta audio muestran el texto que está entre la etiqueta. Es soportado por Chrome 4.0 en adelante, internet explorer/edge 9.0 en adelante, Firefox 3.5 en adelante, safari 4.0 en adelante y opera 10.5 en adelante.

Ejemplo

        
<!DOCTYPE html>
<html>
  <body>
    <audio controls>
      <source src="BUNBURY – Más alto que nosotros sólo el cielo.mp3" type="audio/mpeg">
    </audio>
  </body>
</html>

<canvas>

La etiqueta <canvas> se utiliza para dibujar gráficos sobre la marcha , a través de secuencias de comandos (generalmente JavaScript). La etiqueta <canvas> es solo un contenedor para gráficos, debe usar un script para dibujar los gráficos. Es soportado por Chrome 4.0, Internet explorer 9.0, Firefox 2.0, Safari 3.1 y Opera 9.0 en adelante.

Ejemplo:

        
<canvas id="myCanvas"></canvas>

<script>
var canvas = document.getElementById("myCanvas");
var ctx = canvas.getContext("2d");
ctx.fillStyle = "#FF0000";
ctx.fillRect(0, 0, 80, 80);
</script>
Your browser does not support the HTML5 canvas tag.

La etiqueta <datalist> es usada para proveer una opción de "autocompletado" en los elementos <input>. El usuario verá una lista desplegable de opciones pre-definidas.

Es soportado por Chrome, Edge, FireFox y Opera. También soporta los atributos globales de HTML.

<dd>

La etiqueta <details> puede ser usada para crear un "widget" interactivo que el usuario pueda abrir y cerrar. Cualquiero tipo de contenido puede ser puesto dentro de la etiqueta.

<picture>

La etiqueta <picture> brinda a los desarrolladores web más flexibilidad para especificar recursos de imagen. El uso más común del elemento <picture> será para la dirección de arte en diseños sensibles. En lugar de tener una imagen escalada hacia arriba o hacia abajo según el ancho de la ventana gráfica, se pueden diseñar varias imágenes para que ocupen más espacio en la ventana gráfica del navegador. El elemento <picture> contiene dos etiquetas diferentes: una o más etiquetas <source> y una etiqueta <img>. El navegador utilizará los valores de los atributos para cargar la imagen más adecuada. El navegador utilizará el primer elemento <source> con una sugerencia coincidente e ignorará las siguientes etiquetas de <source>. El elemento <img> se requiere como la última etiqueta secundaria del bloque de declaración <picture>. El elemento <img> se utiliza para proporcionar compatibilidad con versiones anteriores para los navegadores que no admiten el elemento <picture>, o si ninguna de las etiquetas <source> coincide. El elemento <picture> funciona de manera similar a los elementos <video> y <audio>. Usted configura diferentes fuentes, y la primera fuente que se ajusta a las preferencias es la que se está utilizando.

Ejemplo:

  
<picture>
  <source media="(min-width: 650px)" srcset="img_pink_flowers.jpg">
  <source media="(min-width: 465px)" srcset="img_white_flower.jpg">
  <img src="img_orange_flowers.jpg" alt="Flowers" style="width:auto;">
</picture>

<template>

La etiqueta <template> mantiene su contenido oculto al cliente. El contenido dentro de una etiqueta <template> no se procesará. El contenido puede ser visible y renderizado más tarde usando JavaScript. Use la etiqueta <template> cuando tenga el código HTML que quiera usar una y otra vez, pero no hasta que lo solicite. Para hacer esto sin la etiqueta <template>, debe crear el código HTML con JavaScript para evitar que el navegador rinda el código.Esta desde Chrome 26.0, Internet Explorer 13.0, Firefox 22.0, Safari 9.0 y Opera 15.0 en adelante.

Ejemplo:

  
<template>
  <h2>Flower</h2>
  <img src="img_white_flower.jpg">
</template>

Ejemplo:
